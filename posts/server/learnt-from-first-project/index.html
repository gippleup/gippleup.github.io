My Past and Future

Learnt from first project

May 24, 2020

What I Learnt

Good job

  1. Made solid agreement on redux structure.
  • Before doing our first project, I’ve spent some time for making simple web application and found that once I write and apply dispatches and props for several components, it’s quite hard to change the data structure of redux store, just becuase many parts are affected by a simple change.
  • So, I suggested that we should design store structure in advance for preventing waste of time. It worked.
  1. Imagined possible user interactions thoroughly in advance. It was neccessary to imagine all the possible interactions in advance, because it defines everything.

Mistakes

  1. Didn’t make solid agreement on api.
  • It was mistake. We skimmed through api. As a result, client-side develeopment has slowed down. I should have insisted we need to design api more thoroughly.
  1. Skipped designing codes, result? some redundant code.
  • Checking auth for request is redundant in my code. I had to design each controller in adanvce, find common parts, and handle it with middleware.

Lessons(Server)

  1. Server doesn’t have to subordinate to user interaction.
  • Server need its own interface to communicate with database. When server tries to make seperate apis for each client request, it gets more complex to balance between database and client.
  1. All the possible errors should be trackable.
  • If you want to skip error handling, think about the gloomy future which you scoll through the error messages and can’t find relavant line.
  1. Use sql syntax for simple structured data, use ORM for nested data.
  • If you want to nest each table in a data, it’s fair to use ORM. If you want to structure your data in simple dimension, you can use sql syntax.

Lesson(General)

  1. For teamwork, focus on the joinned part between two parties.
  • If we have discussed about api thoroughly, client-side could earn much more time.
  1. If you should write more than 10 lines of codes, don’t skip design.
  • Find common parts and put it in seperate lines.
