My Past and Future

News.pick.io

Apr 29, 2020

Motivation

  1. My brother asked me to make such app, with slight mocking that I would not be able to make such app. I answered “Well, that’s easy peasy”. I started making app.
  2. I wanted to practice redux. ‘News-pick? There would be quite a lot of data-flow, good.’

User interaction / Data flow

Search

Input Keyword
- User click input box and type keyword.
/* Inputs are saved to local state. */
- When click ok button,
/* Fetch News for keywords. */
  1. Keyword is added to keyword list 
  /* Keywords are add to global state */
  2. New keyword block is generated
  /* News blocks are added to global state */
  3. Existing keyword block is updated
  /* News blocks are updated */
  /* Use url as item id */
  4. Press list is updated
  /* Press list is added to global state */
  /* Press that has not been touched,
      will be automatically updated. */
  /* Press that has been touched,
      will remain filtered if there is news,
      will be inactive when there's no news. */
Manual Update
- User click update button,
  then existing keyword news are updated.
  /* 1. Fetch News for keywords. */
  /* 2. News blocks are updated. */
Set Interval for Update
- User set intervals using button
/* Interval is saved to local state */
- User click ok button for completing setting
/* Interval is saved to global state */
Set Filter for News
- User can toggle each keyword.
/* Each toggle is saved to global state[keyword] */
- User can toggle each press.
/* Each toggle is saved to global state[press] */
- User can toggle each tags.
/* Each toggle is saved to global state[tags] */
- User can toggle 'show all' button for press/keyword.
/* Each toggle becomes useless and only global filter rules */
/* When deactivate show all, each toggle is activated */
- User can toggle time filter.
/* Time filter is saved to global state */

News

Read News for Keywords
- User toggle tag, then it shows related news.
/* Each news block store spread state */
- User can see time, press, originallink, etc.
/* Each news item receives such props[State => prop] */
Toggle News
- User can toggle complete.
/* Completed items are saved to global state[Store: completed] */
- User can toggle hide.
/* Hidden items are saved to global state with tags[Store: hidden] */
- User can toggle later.
/* Items are saved to global state[Store: later] */
- User can put temporal tags.
/* Tags are saved to target item[NewsItem: tags], and global state[Store: tags] */

Subscription

Add Subscription Email
- User can add temporal subscription email,
  by typing their email and purpose.
  /* Email is saved to local state[EmailList: emails] */
Delete Subscription Email
- User can delete subscription email,
  by clicking delete button.
  /* Email is deleted from local state[EmailList: emails] */
