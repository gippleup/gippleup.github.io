Server-related Concepts

Apr 06, 2020

In short

  1. There is the Web, which is accessible through internet. It’s a giant information system which contains all the resources over the globe and each resources are linked by hyperlinks and URLs.
  2. Internet network of networks which enables us to access to WWW resources by using HTTP or HTTPS(This is called protocol). But itself is not user-friendly. What enalbes us to access the Web easily is web browsers.
  3. Browser is an application for accessing information on the World Wide Web. When a user reqests a particular website, the web browser retrieves the necessary content from a web server and then displays the resulting web page on the user’s device. (Browser parses our html, css, js file into web content)
  4. Web server is server software, or hardware dedicated to running this software, that can satisfy client requests on the World Wide Web. A web server processes incoming network requests over HTTP and other related protocols.
  5. Client is a piece of computer hardware or software that accesses a service made available by a server.
  6. Protocol is a system of rules that allow entities to transmit information via any kind of variation of devices.
  7. HTTP is an application protocol for the World Wide Web, where hypertext documents include hyperlinks to other resources that the user can easily access, for example by a mouse click.
  8. Hypertext is text with hyperlinks.
  9. Hyperlink, or a link, is a reference to data that the user can follow by clicking or tapping.

Server-related concepts

  1. API is a computing interface to a software component or a system, that defines how other components or systems can use it. It defines the kinds of calls or requests that can be made, how to make them, the data formats that should be used, the conventions to follow, etc. For example, this is a pokemon go API.
  2. AJAX is a set of web development techiniques using many web technologies on the client side to create asynchronous web applicatons. With Ajax, server can work in the background without interfering with the display and behavior of the existing page.

Browser

Components of browser

  1. The user interface: this includes the address bar, back/forward button, bookmarking menu, etc. Every part of the browser display except the window where you see the requested page.
  2. The browser engine: marshals actions between the UI and the rendering engine.
  3. The rendering engine : responsible for displaying requested content. For example if the requested content is HTML, the rendering engine parses HTML and CSS, and displays the parsed content on the screen.
  4. Networking: for network calls such as HTTP requests, using different implementations for different platform behind a platform-independent interface.
  5. UI backend: used for drawing basic widgets like combo boxes and windows. This backend exposes a generic interface that is not platform specific. Underneath it uses operating system user interface methods.
  6. JavaScript interpreter. Used to parse and execute JavaScript code.
  7. Data storage. This is a persistence layer. The browser may need to save all sorts of data locally, such as cookies. Browsers also support storage mechanisms such as localStorage, IndexedDB, WebSQL and FileSystem.
