Data Structure: Queue n Stack

Mar 19, 2020

For managing task, queue n stack is useful.

Intro

Each data structures are opt for different tasks. This page illustrates each data structure’s concept, advantage, and disadvantage.

Basic Data Structure

Queue and Stack are maybe one of most simple data structure. Queue is used for FIFO(First In First Out - Imagine lining up for ice cream) and Stack is used for LIFO(Last In First Out - Imagine stacking dishes)

Queue

(img from programiz)

Belows are common methods included in queue.

  1. Enqueue: Adds an item to the queue. If the queue is full, then it is said to be an Overflow condition – Time Complexity : O(1)
  2. Dequeue: Removes an item from the queue. The items are popped in the same order in which they are pushed. If the queue is empty, then it is said to be an Underflow condition – Time Complexity : O(1)
  3. Front: Get the front item from queue – Time Complexity : O(1)
  4. Rear: Get the last item from queue – Time Complexity : O(1)

- From GeeksForGeeks


Queue, written in JS

const Queue = function () {
    this.storage = {};
    this.count = 0;
};

Queue.prototype.enqueue = function (value) {
    this.storage[this.count] = value;
    this.count ++;
    console.log(this.storage);
}

Queue.prototype.dequeue = function () {
    if (this.count <= 0) {
        return;
    }
    let result = this.storage[0];
    for (let i = 0; i < this.count - 1; i++) {
        this.storage[i] = this.storage[i+1];
    }
    delete this.storage[this.count - 1];
    this.count --;
    console.log(this.storage);
    return result;
}

Queue.prototype.size = function () {
    return this.count;
}

Queue.prototype.front = function () {
    return this.storage[0];
}

Queue.prototype.rear = function () {
    return this.storage[this.count - 1]
}

Advantage of Queue

When there is no need for modifying values in queue, queue can be useful. Basic queue methods are time-efficient. enqueue, size, front, rear methods show O(1), and dequeue shows O(n) time-efficiency. When a task should be performed in order and doesn’t require much modification on data structure, you may consider using queue.

Disadvantage of Queue

It’s not opt for modifying data in itself because a data is saved without an id or specific name. For modifying a data, you need to search for all data in itself(if the data you want to modify sits in index n, you need to compare values n times).

Stack

(img from programiz)

Belows are common methods included in stack.

  1. push(Object element) : Pushes an element on the top of the stack.
  2. pop() : Removes and returns the top element of the stack. An ‘EmptyStackException’ exception is thrown if we call pop() when the invoking stack is empty.
  3. peek() : Returns the element on the top of the stack, but does not remove it.
  4. empty() : It returns true if nothing is on the top of the stack. Else, returns false.
  5. search(Object element) : It determines whether an object exists in the stack. If the element is found, it returns the position of the element from the top of the stack. Else, it returns -1.

- From GeeksForGeeks

Stack, written in JS

const Stack = function () {
    this.storage = {};
    this.count = 0;
}

Stack.prototype.push = function (value) {
    this.storage[this.count] = value;
    this.count ++;
}

Stack.prototype.pop = function () {
    if (this.count === 0) {
        return;
    }
    let result = this.storage[this.count - 1];
    delete this.storage[this.count - 1];
    this.count --;
    return result;
}

Stack.prototype.size = function () {
    return this.count;
}

Advantage n Disadvantage?

Well, it's very similar to queue. What differs stack from queue is that queue let out which came first, stack let out which came last.
