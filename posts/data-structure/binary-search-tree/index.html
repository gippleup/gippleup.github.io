Data Structure: Binary Search Tree(BST)

Mar 23, 2020

Binary Search is stable and fast.

Concept

(img from programiz)

Binary search tree, as its name implies, does arrange an item automatically when we add it.

If there are 10,000 items in an array, the minimum time-complexit will be O(n). However, if items can be arranged in binary search tree, the tree would have 13 as depth. And this will reduce the calculation count to 13 from 10,000.

In ideal setting, 10,000 calculation can be replaced with 13 calculation, with binary search tree data structure.


Belows are common methods included in binary search tree.
  1. addChild: Add a child tree node to existing tree.
  2. contains: Check if an item exist in the tree.

Binary Search Tree, written in JS

const BinarySearchTree = function (value) {
    let instance = {};
    instance.value = value;
    instance.left = null;
    instance.right = null;

    instance.insert = function (value) {
        if (value < instance.value) {
            if (instance.left === null) {
                instance.left = BinarySearchTree(value);
            } else {
                instance.left.insert(value);
            }
        } else {
            if (instance.right === null) {
                instance.right = BinarySearchTree(value);
            } else {
                instance.right.insert(value);
            }
        }
    }


    instance.contains = function (value) {
        if (value === instance.value) {
            return true;
        } else {
            if (value < instance.value) {
                if (instance.left !== null) {
                    return instance.left.contains(value);
                }
            } else {
                if (instance.right !== null) {
                    return instance.right.contains(value);
                }
            }
        }
        return false;
    }

    instance.depthFirstLog = function (cb) {
        if (instance !== null) {
            cb(instance.value);
            if (instance.left !== null) {
                instance.left.depthFirstLog(cb);
            }
            if (instance.right !== null) {
                instance.right.depthFirstLog(cb);
            }
        }
    }

    return instance
};

Advantages

  • It can reduce search time exponentially, as like tree structure.
  • The calculation time required is static(in other words, stable). For example, when 10,000 items are represented in tree structure which has 100 nodes for each subtree, it requires 4~40 calculation. Howevert, when the same items are represented in binary search tree, it would require 13 calculation at maximum.
