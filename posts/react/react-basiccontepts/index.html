React - Basic Concepts

Apr 15, 2020

React itself

React is very module-friendly.

Without React
<div>
  <h1>Hello, world!</h1>
  <h2 id="clock">Here goes current time.</h2>
</div>

function tick() {
  let renderTarget = document.querySelector('h2#clock')
  renderTarget.textContent = 
  `It is ${new Date().toLocaleTimeString()}.`
}
setInterval(tick, 1000);
With React
function tick() {
  const element = (
    <div>
      <h1>Hello, world!</h1>
      <h2>It is {new Date().toLocaleTimeString()}.</h2>
    </div>
  );
  ReactDOM.render(element, document.getElementById('root'));
}

setInterval(tick, 1000);

Advantage using React

1. Less 'id' and 'class' tag

When html element and javascript are seperated, in other words, when target element and functions are in seperate location, it requires a way to call target element in function.

In pure js,

  1. Set id or class to target element
  2. Select target element using querySelector
  3. Execute function

This can be troublesome when target node has many child nodes which requires control for each. We may have to do something like this.

let functionality = function() {
  let select = (ele, query) => ele.querySelector(query);
  let parentEle = select(document, '#parent');
  //do something with parentEle

  let selectChild = (query) => select(parentEle, query);
  let childEle1 = selectChild('#child1');
  //do something with childEle1

  let childEle2 = selectChild('#child2');
  //do something with childEle2

  let childEle3 = selectChild('#child3');
  //do something with childEle3
}

If target element has complex structure, it becomes hard to make things clear, in turn, makes it hard to manage our function.

let reacModule = function() {
  somthing1() {}
  somthing2() {}
  somthing3() {}
  return (
    <div>
      Parent
      <div onClick={this.something1}>Child1</div>
      <div onClick={this.something2}>Child2</div>
      <div onClick={this.something3}>Child3</div>
    </div>
  )
}

However, in react, we can make things more clear, in turn, helps us to manage our app more easily by

  1. setting html structure clearly in the render section,
  2. seperating required methods for element.
2. Easy to manage a module

Rendering

Target to render

class App extends React.Component {
  render() {
    return <div>This is {this.props.appname}.</div>
  }
}
In JSX
ReactDOM.render(
  <App appname="React-App"/>,
  document.getElementById('root')
)
In RectDOM.render()
ReactDOM.render(
  React.createElement(Hello, {appname: 'root'}, null),
  document.getElementById('root')
)
At a glance
<App appname="React-App">
//===These two are the same===
React.createElement(App, {appname: 'React-App'}, null)

Temp-Rendering for wrapping

Sometimes, you may want to render a component temporarily as HTML. For example, if you want to import html element of a component, you can render it temporarily and select your desired element.

const tempComponent = (props) => {
  return (
    <div id={parent}>
      <div id={child1}>Child1</div>
      <div id={child2}>Child2</div>
      <div id={child3}>Child3</div>
    </div>
  )
}

//1) Make Wrapper
let wrapper = document.createElement('div');
//2) Render it as a text
let textRender = ReactDOMServer.renderToString(tempComponent)
//3) Insert text-render to Wrapper
wrapper.innerHTML = textRender;
//4) Done. Now, you can use the component.
let tempRendered = wrapper.children[0];
let targetEle = tempComponent.querySelector('div#child3');
let targetValue = targetEle.textContent; //Child3

Just Remember this

  1. React is useful for developing front-end module, becuase you can pack everything(css, script, data-flow) in a single component.
  • Refs are useful for controling interactions between parts of a component.
  1. In react, data flows from top to bottom using props. If you want to change data of upper component, pass data changing methods to lower component.
  1. React is useful, but with redux it becomes much efficient because you can control data flow more easily.
