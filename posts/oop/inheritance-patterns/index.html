Inheritance Patterns in JS

Mar 25, 2020

What is Inheritance?

Inheritance makes it easy to reuse codes.

const Book = function (category) {
  this.category = category;
}

const Novel = function (genre) {
  Book.call(this, 'novel');
  this.genre = genre;
}

Object.setPrototypeOf(Novel.prototype, Book.prototype);

Novel.prototype.genreInfo = function () {
  console.log(`The book's genre is ${this.genre}.`)
}

Book.prototype.categoryInfo = function () {
  console.log(`The book's category is ${this.category}`)
}

let myNovel = new Novel ('thriller');

myNovel.genreInfo() //The book's genre is thriller
myNovel.categoryInfo() //The book's category is nove

Inheriting properties from another class or classes to create a class. Above codes shows inheriatnce from Book class to Novel class. Novel class has inherited ‘category’ property and ‘categoryInfo’ method from Book class.

Why does it matter?

Object oriented programming is about constructing reality by using objects. Object inheritance make things more structured. Let’s say we have class a ‘thing’ and it has all properties that required to build everything in the universe. Then it should own almost infinite numbers of properties. If it does so, it would be grotesque to input all the parameters…

const Thing = function (
    mass, component, volume,
    hasLife, isPlanet, isWater,
    isHuman, habit, hobby,
    flavour, smell, hasAuditory,
    hasVisual, hasBrain, ...
    ) {
    ...
}

let John = new Thing(
    156, [Oxygen, Carbon, Hydrogen, Nitrogen, ...],
     ...
    )

At this point, we need inheritance.

With inheritance, inputing parameters when declaring a thing with ‘thing’ class can be more simple. Like This.

const Thing = function ('few param') {}
const LivingThing  = function ('few param') {
    //...Inherit Thing
}
const Animal = function ('few param') {
    //...Inherit LivingThing
}
const Vertebrate = function ('few param') {
    //...Inherit Animal
}
const Human = function('few param') {
    //...Inherit Vertebrate
}

With inheritance we can set some parameters of parent class as default value that we want and don’t have to invent similar method again. (If you don’t to inherit some properties of parent class, you can delete them from prototype just like deleting a property in an object.)

In short, inheritance is about inputting parameters automatically and importing wanted methods from parent class. With proper inheritance, it becomes easier to understand written codes and build new instance.

Inheritance Patterns

Two things to bear in mind is these. When an object is created using Object.create(proto), the prototype of the object is set to proto. Object.create(proto) is different from Object.assign(object, proto) in that Object.assign(object, proto) doesn’t set prototype.

const Drink = function (type) {
  let drink = Object.assign(Drink.prototype);
  return drink;
}

Drink.prototype = {
  info () {
    console.log('This is a drink')
  }
}

let myDrink = Drink('water')
myDrink.info(); //'This is a drink'
console.log(Drink.prototype.isPrototypeOf(myDrink)) //false

//Though the method in side Drink.prototype is assigned,
//it doesn't mean that the prototype of myDrink is Drink.

As said right before, Object.assign(object, proto) can’t set prototype. Then how can we inherit from other classes? There are two methods. One is using Object.setPrototypeOf(obj, proto), and the other is usidng ES6 Class Syntax(extends).

  1. let obj = Object.create(proto) sets prototype of obj as proto.
  2. Object.setPrototypeOf(obj, proto) sets internal prototype of obj as proto.
  • PseudoClassical Pattern; using this, Function.call(), Object.setPrototypeOf(), new;
  • ES6 Class Pattern; using extends, super
  • Object Pattern; using Object, Object.create(), Object.setPrototypeOf()
  • Functional + Object Pattern; using Function, Object.create(), Object.setPrototypeOf()

PseudoClassical Pattern

const Drink = function (type) {
  this.type = type;
}

Drink.prototype.drinkInfo = function () {
  console.log(`The type of this drink is ${this.type}.`)
}

const Soda = function (name, type) {
  Drink.call(this, 'soda');
  this.name = name;
  this.type = type;
}

Object.setPrototypeOf(Soda.prototype, Drink.prototype);
//Or
//Soda.prototype = Object.create(Drink.prototype);
//Soda.prototype.constructor = Soda;

Soda.prototype.sodaInfo = function () {
  console.log(`The type of this soda is ${this.type}.`)
  console.log(`The name of this soda is ${this.name}.`)
}

let mySoda = new Soda('sprite', 'cider');
mySoda.drinkInfo()
//The type of this drink is cider.
mySoda.sodaInfo()
//The type of this soda is cider.
//The name of this soda is sprite.

ES6 Class Pattern

class Animal {
  constructor(species) {
    this.species = species;
  }
  get info () {
    console.log(`The species of this creature is ${this.species}`)
  }
}

class Dog extends Animal{
  constructor(name, breed) {
    super('dog');
    this.name = name;
    this.breed = breed;
  }
  get info () {
    super.info
    console.log(`The name of this dog is ${this.name} and the breed is ${this.breed}`)
  }
}

class Human extends Animal {
  constructor(name, sex) {
    super('human')
    this.name = name;
    this.sex = sex;
  }

  get info () {
    super.info
    console.log(`The name of this human is ${this.name} and sex is ${this.sex}`)
  }

  static kiss (obj1, obj2) {
    let isHuman1 = Human.prototype.isPrototypeOf(obj1);
    let isHuman2 = Human.prototype.isPrototypeOf(obj2);
    if (isHuman1 && isHuman2) {
      console.log(`Human who has name ${obj1.name} has french kissed with another huuman whose name is ${obj2.name}.`)
    } else {
      let humanName = Array.from(arguments).reduce((acc, ele) => {
        if(Human.prototype.isPrototypeOf(ele)) {acc = ele} return acc}, {}).name;
      if (humanName) {
        console.log(`Hey ${humanName}! french kissing between different species are inhibited by federal law!`)
      } else {
        console.log(`There is no human`)
      }
    }
  }

  walk () {
    console.log(`Human ${this.name} started to walk.`)
  }
}


let john = new Human('john', 'male');
let marry = new Human('marry', 'female');
let spark = new Dog('spark', 'poodle');

john.info

Object Pattern

const Planet = {
  create(type) {
    var planet = Object.create(Planet.prototype);
    planet.type = type;
    return planet;
  },
  prototype: {
    spin() {
      console.log(`The type of this planet is ${this.type}.`)
      console.log(`A planet is spinning. Bing Bing.`)
    }
  }
}

const SolarPlanet = {
  create(name) {
    var solarPlanet = Object.create(SolarPlanet.prototype);
    Object.assign(solarPlanet, Planet.create('solar planet'));
    solarPlanet.name = name;
    return solarPlanet;
  },
  prototype: {
    info() {
      console.log(`The name of this planet is ${this.name}.`);
    }
  }
}

Object.setPrototypeOf(SolarPlanet.prototype, Planet.prototype);
//Or
//SolarPlanet.prototype = Object.create(Planet.prototype);
//SolarPlanet.prototype.constructor = SolarPlanet;

let mySolarPlanet = SolarPlanet.create('earth');
mySolarPlanet.spin();
mySolarPlanet.info();

Functional + Object Pattern

const Exercise = function (type) {
  var exercise = Object.create(Exercise.prototype);
  exercise.type = type;
  return exercise;
}

Exercise.prototype = {
  exerciseInfo() {
    console.log(`The type of this exercise is ${this.type}`)
  }
}

const Cardio = function(name, calorie) {
  var cardio = Object.create(Cardio.prototype);
  Object.assign(cardio, Exercise('cardio'))
  cardio.name = name;
  cardio.calorie = calorie;
  return cardio;
}

Cardio.prototype = {
  cardioInfo() {
  console.log(`The name of this exercise is ${this.name}`)
  console.log(`The average calories consumed is ${this.calorie}`)
  }
}

Object.setPrototypeOf(Cardio.prototype, Exercise.prototype);
//Or
//Soda.prototype = Object.create(Drink.prototype);
//Soda.prototype.constructor = Soda;

let myCardio = Cardio('swimming', 413);
myCardio.exerciseInfo()
myCardio.cardioInfo()

Evaluation

1. Pseudocalssical Pattern

The drawback of Pseudocalssical Pattern is that it requires using ‘this’ keyword which sometimes can make things seem more complex than it is. However, javascript system Objects follow pseudoclassical patterns. That is, being accustomed to this pattern is necessary.

This method has an advantage in that we can seperate each methods or prototype property by using Class.prototype.methodName = () {}.

2. ES6 Class Pattern

With class-constructor pattern, methods and constructing parts of class should be set inside class declaration. This can help arranging code. Another advantage followed by using ES6 pattern is that we don’t have to use ParentClass.call(parm1, parm2, parm3) for inheriting properties and method. We can simply use super(parm1, parm2, parm3), and super is handy because you can access to parent class by using somethig like super.methodName or super.propertyName.

3. Object Pattern

If you don’t want to use ‘this’ keyword, you can use Object Pattern. Drawback of this method is that you can’t access to parent classes through proto.

If you use Object Pattern example from above, you would see below log when you console.log(youObjectName). What you can see is, the proto is not set as parent class. This is not good because you cannot explicitly see what the parent class of you instnace.

The advantage of this pattern is that you don’t have to use this. But, with my opinion, Using ES6 of Pseudofunctional Pattern is better because you can see What is the parent class of the instance. Think about a situation where you have 20,000 lines of codes and don’t know the parent class of your target instance.

4. Object + Functional Pattern

This pattern is similar to 3rd pattern. But it requires using this keywords.

Use ES6 Pattern or Pseudo-classical Pattern

In my opinion, setting __proto__ and prototype.constructor clearly is better than throwing out this keyword because you can easily look up parent class when you need. But, it would be better to know Object Pattern and Object+Functional Pattern for understanding other’s codes.
