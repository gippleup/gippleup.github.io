Mar 19, 2020

What is OOP?

OOP makes it easier to manifest hiearchical structure.

OOP can be descibed with class and instance. Class creates instances and instance is a product of class.

For real-life example, a tomato juice is a instance of juice, and juice is a class for tomato juice.

In short, OOP is about structuring things.

Let’s say we are building a webApp which provides all sorts of UI buttons.

When building this, we would make many kinds of buttons - like popup button, submit button, animating button, etc.

When the number of buttons are not big, there would be no problem even if we build each buttons with seperate functions.

var submitButton = {
    onClick: 'do something',
    color: 'red',
    size: '10px',
}

var animatingButton = {
    onClick: 'do something',
    color: 'blue',
    size: '12px',
}

//...continues

However, when the projet becomes bigger and more buttons are needed, declaring each buttons independently would be a problem.

Let say, you have 100 buttons and users complain about something that is common for all the buttons.

Without OOP, you would have to fix all the buttons one by one, which would requires your backbones to break.

OOP can save you

Meanwhile, if you have built buttons based upon OOP, you would be able to fix the problem by fixing one function - the class function.

There are many pages that explaining about OOP in detail. Like this

Think structure

function Button (name, type, size) {
    this.name = name;
    this.type = type;
    this.size = size;
}

function AnimatedButton (name, type, size, animation, duration) {
    Button.call(this, name, type, size);
    this.animation = animation;
    this.duration = duration;
}

let myButton = new AnimatedButton('shrinkButton', 'animated', '10px', 'shrink', '1s');

console.log(myButton);
//======================//
//     console said     //
//======================//
// AnimatedButton {     //
// name:"shrinkButton", //
// type:"animated",     //
// size:"10px",         //
// animation:"shrink",  //
// duration:"1s"        //
// }                    //
//======================//

We can manage all kinds of buttons with Button class, but if we want more delicate control, we would have to subdivide the category - like AnimatedButton, SubmitButton, etc.

For applying OOP effectively, understanding about project structure should preceed.
